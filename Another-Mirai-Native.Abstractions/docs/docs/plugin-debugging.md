# 插件调试

## 🔎 附加进程

框架采用多进程架构，因此在附加调试器时需要先区分主进程和插件进程。在附加进程列表中，你会看到一组名称为 `Another-Mirai-Native.exe` 的进程，其中只有一个是主进程，其余都是插件进程。

![进程列表](/images/AttachProcess.png)

要判断某个插件进程对应的是哪个插件，可以查看“命令行”一列。继续向后查看参数后，通常可以看到插件加载路径，从而辨别该进程对应的插件。

![辨别插件](/images/ProcessCommandLine.png)

## 🐞 调试选项

在 `conf/Config.json` 中启用 `DebugMode` 和 `DebugLazyLoad` 后，插件进程会在启动完成、调用启动事件之前暂停等待，此时用户可以主动拉起调试器，从而获得更早的调试时机。

![启动调试器](/images/LaunchDebugger.png)

> ⚠️ **注意**：使用这种调试模式时，建议同时调大 `LoadTimeout` 和 `PluginInvokeTimeout`，避免因为调试时间过长而被框架判定为启动超时并终止进程。
>
> 💡 **提示**：建议调试时只启用一个插件，避免同时弹出过多“启动调试器”请求窗口。
>
> 💡 **提示**：启用 `DebugMode` 后，框架会尝试新建命令行窗口并输出 RPC 相关细节。其中很多日志并不是插件开发时必须关注的信息，可以先忽略。
